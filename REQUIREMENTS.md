# 激光数据标注与训练集生成工具 — 功能需求文档

## 概述

上位机工具 (Python + PyQt6)，将 MCXN947 采集的窄表 CSV 导入、人工标注物理量含义、按设备类型分组导出为 ML 训练用宽表 CSV。

## 核心流程

```
MCXN947 U盘 → data_0001.csv (窄表)
        ↓
   [导入] 拖拽或浏览选择 CSV
        ↓
   [发现] 扫描 {node_id, slot, tc} 组合
        ↓
   [标注] 用户为每个 tc 选择物理量名称 + 单位
        ↓
   [模板] 保存设备类型模板 → 下次同类设备自动标注
        ↓
   [导出] 按设备类型分组的宽表训练 CSV
```

## 功能模块

### 1. 数据导入

- 支持拖拽 CSV 文件或文件夹批量导入
- 预览窗口: 表格显示前 100 行, 统计 node_id/slot/tc 分布
- 自动去重 (按 uptime 去重, 可选保留首次/末次)

### 2. 自动发现

扫描窄表, 统计每个 `{node_id, slot}` 下出现了哪些 `tc`(typecode):

```
设备 2, Slot 0:  tc=[2, 5, 24]        ← 温度+TEC
设备 2, Slot 4:  tc=[]                 ← 空 (FPGA, 不采集)
设备 4, Slot 0:  tc=[2, 3, 4]         ← 电流+电压+功率
设备 4, Slot 1:  tc=[2, 3, 4]         ← 同上
设备 4, Slot 5:  tc=[]                 ← 空

自动分类:
  设备 2 → 疑似 S001 种子源 (有温度/TEC, 无电流)
  设备 4 → 疑似 BC01 电流板 (有电流/电压/功率, 无温度)
```

### 3. 物理量标注界面

对每个 `{设备类型, tc}` 组合, 用户从下拉列表选择物理量名称:

| tc 值 | 所属 FUNC | 可选标注名 |
|-------|----------|-----------|
| 1 | RPTCURR / RPTTEMP | C_TARGET(电流目标) / T_TARGET(温度目标) |
| 2 | RPTCURR / RPTTEMP | C_READ(电流实际值) / T_READ1(T1实际温度) |
| 3 | RPTCURR | C_VREAD(LD电压) |
| 4 | RPTCURR | C_PSUMP(LD功率) |
| 5 | RPTTEMP | T_TEC(TEC PWM强度) |
| 21 | RPTTEMP | TP_TREAL(T1温度) |
| 24 | RPTTEMP | TP_TEC(TEC PWM) |
| 31 | RPTCURR | TP_CURR(LD电流) |
| 32 | RPTCURR | TP_LDV(LD电压) |
| 33 | RPTCURR | TP_LDPS(LD功率) |

同时设置:
- **单位**: 自动填充 (mA/mV/mW/°C/无量纲), 可手动修改
- **数据类型**: 实际值 / 目标值 / 其他
- **是否纳入训练特征**: 勾选框

### 4. 目标值配对

对于标注为"实际值"的物理量, 自动匹配同 slot 下的"目标值"计算偏差:

```
slot=0, tc=2(C_READ 实际电流), val=998.5
slot=0, tc=1(C_TARGET 目标电流), val=1000
→ 偏差 = 998.5 - 1000 = -1.5
→ 新增列: C_DEV_mA = -1.5
```

PD 功率衰减率 (预留):
```
PWR_mW 实际 / PD_PWR_mW 反馈 → 衰减率
```

### 5. 模板系统

标注完成后保存为模板文件 (JSON):

```json
{
  "device_type": "S001_Seed_Source",
  "slots": [
    {
      "slot": 0,
      "measurements": {
        "2": {"name": "T1_ACTUAL", "unit": "C", "type": "actual"},
        "5": {"name": "TEC_PWM", "unit": "", "type": "actual"},
        "1": {"name": "T1_TARGET", "unit": "C", "type": "target"}
      }
    }
  ]
}
```

下次导入时自动匹配模板——如果新数据的 `{node_id, slot, tc}` 组合与已保存模板完全一致, 自动套用标注。

### 6. 宽表导出

按**设备类型分组**导出训练用 CSV:

```
S001_Seed_Source_slot0.csv:
  uptime,node_id,T1_TARGET_C,T1_ACTUAL_C,T1_DEV_C,TEC_PWM
  12345,2,25.0,25.1,0.1,0.75
  12650,2,25.0,25.2,0.2,0.74

BC01_Current_Board_slot0.csv:
  uptime,node_id,C_TARGET_mA,C_ACTUAL_mA,C_DEV_mA,V_ACTUAL_mV,PWR_mW
  12345,4,1000,998,-2,2450,2445
  12650,4,1000,997,-3,2460,2453
```

**列命名规则**: `{物理量名}_{单位}` (如 `T1_ACTUAL_C`, `LD_CURR_mA`)

**缺失值**: 空字段用 NaN 或留空

**时间对齐**: 所有行按 uptime 排序, 为目标值前向填充 (forward fill)

### 7. 数据预览与统计

- 时序折线图: 选择 node_id + slot + 测量量, 绘制时间序列
- 目标值 vs 实际值对比图: 同一 slot 的目标值、实际值、偏差三条线
- 统计摘要: 均值/方差/最大偏差/数据点数/时间跨度

### 8. PD 功率预留

K64 尚未实现 PD 功率采集, 但在工具层面预留:
- 标注列表中已有 `PD_PWR` 选项
- 导出的宽表中自动包含 `PD_PWR_mW` 列 (当前值=NaN)
- K64 固件升级实现 PD 功率后, 旧数据自动补充

## 技术栈

| 层 | 技术 |
|----|------|
| GUI | PyQt6 |

| 图表 | matplotlib (嵌入 PyQt) |

| 打包分发 | PyInstaller (单文件 exe) |
